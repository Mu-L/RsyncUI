Recommended Changes

Each change below maps to a specific area of the mockup. The goal is to improve scannability, reduce visual noise, and surface the most important information faster — while keeping the macOS-native feel.

1 Sidebar: section grouping + badges

Group sidebar items under labeled sections (Actions, Tools, Management) to create visual hierarchy. Add count badges to show tasks needing attention at a glance — e.g. "2" on Synchronize means 2 tasks have pending changes.

Current: flat list of 4-5 items, no grouping, no status indicators Proposed: 3 sections with headers, contextual badges, better iconography

2 Profile picker: promoted + labeled

Move the profile picker to a dedicated area at the top of the sidebar with a clear "Profile" label. The current implementation blends into the sidebar when no profiles exist. A persistent labeled dropdown communicates the active context immediately.

Current: unlabeled Picker that hides when only Default exists Proposed: always-visible labeled dropdown with clear section boundary

3 Toolbar: labeled primary actions

Replace icon-only toolbar buttons with labeled buttons for the 3 primary actions (Estimate, Synchronize, Estimate & Sync). Icon-only buttons require learning; labels make the app self-documenting. Keep secondary tools (Charts, Log, Schedule) in a "more" overflow or behind the existing toolbar toggle.

Current: 8+ icon-only toolbar buttons (wand, play, clear, magnifying glass, hare, chart, calendar, doc, download, shield) Proposed: 3 labeled primary buttons + 1 reset + search field. Secondary tools via menu/toggle

4 Table: status dots instead of text Action column

Replace the text "Action" column (synchronize / syncremote / snapshot) with a compact colored status dot. Green = synced recently, orange = stale (configurable days threshold), grey = halted. The task type (snapshot, syncremote) can appear as a subtle inline badge next to the task name only when it differs from the default "synchronize."

Current: "Action" column shows "synchronize" text repeatedly, truncated at small widths Proposed: 8px colored dot + optional type badge. Saves ~80px column width

5 Table: relative time + date as secondary

Show "Last Sync" as relative time ("15 hrs ago", "14 days ago") with the absolute date below in a smaller size. Relative time is faster to scan. Stale tasks (beyond the configured threshold) get red text to draw attention.

Current: two separate columns "Time last" and "Date last" with truncated values ("15.4 hou...", "10 May 2026...") Proposed: single "Last Sync" column with relative time + small absolute date. Stale = red

6 Table: inline progress for active syncs

When a task is actively syncing, show a compact inline progress bar in the status column and percentage in the time column. This replaces the current approach of showing separate "%" and "Num" columns that appear/disappear.

Current: two extra columns ("%" with ProgressView, "Num" with fraction) that toggle visibility Proposed: progress bar replaces status dot; percentage replaces relative time. No column changes

7 Inspector: segmented control instead of TabView

Replace the macOS TabView (with its large tab bar) with a compact segmented control at the top of the inspector. This saves vertical space and looks more integrated with the inspector panel aesthetic. Group form fields under clear section headers (Identity, Folders, Remote) instead of flat label-field pairs.

Current: full TabView with 4 tabs taking ~35px header + unlabeled form fields Proposed: compact segmented control (~28px) + grouped form sections with headers

8 Inspector: folder browse buttons

Add small browse buttons next to folder path inputs. The current UI has blue squares but they're not discoverable. An inline folder icon button next to each path field makes the affordance clearer.

Current: blue square icons after path fields, purpose unclear Proposed: labeled folder-browse button with clear icon, consistent sizing

9 Halted tasks: visual dimming

When a task is halted, dim the entire row (opacity 0.5) and show "halted" in the time column instead of stale date info. This makes it immediately clear which tasks are inactive without needing to read the Action column.

Current: red stop icon in Action column, rest of row looks normal Proposed: entire row visually dimmed, grey status dot, "halted" label

10 Overall: tighter column layout

Reduce from 8 columns to 6 in Synchronize view (status dot, Task, Source, Destination, Server, Last Sync). In Tasks/Edit view, drop Last Sync entirely since you're editing, not monitoring. This prevents the chronic truncation visible in both screenshots.

Current: 8 columns (%, Num, Synchronize ID, Action, Source, Destination, Server, Time, Date) — many truncated Proposed: 6 columns in Sync view, 5 in Tasks view. All comfortably fit at 1200px
